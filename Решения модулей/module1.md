# Этапы выполнения модуля 1

### Раздел 1 (подготовка данных)

1. Открыть предоставленные данные для импорта в Excel
2. Выбрать пункт экспорта:
 
> [!NOTE]
> **Важно:** Выбор может зависеть от того есть ли активация Office.
> - **Нет:** будет только **Текст Юникод**, вариант **CSV UTF-8** будет отсутствовать.
> - **Да:** будет и **Текст Юникод** и **CSV UTF-8**, выбор будет за вами.

---

#### 1. **Текст Юникод (разделитель пробел)**
  
**Пояснения:**
  - В данном случае файл записывается в .txt с кодировкой **`UTF-16`** её необходимо изменить с помощью **Notepad++**  на **`UTF-8`**. И при импорте в `pgAdmin 4` выбрать разделитель **`[Tab]`**
  
**Проблемы:**
  - В данном формате Excel записывает пустые строки в конец файла, они вызовут ошибку при импорте, поэтому их **необходимо убрать**
  - Так же в файле записаны заголовки столбцов, `pgAdmin 4`, имеет баг, из за которого он не читает заголовки (только для **`*.txt`**), поэтому их **необходимо убрать**

---

#### 2. **CSV UTF-8 (разделитель запятая)**
  
  **Пояснения:**
  - В данном формате файл записывается в UTF-8, это корректная кодировка как для Windows (если включена [поддержка](../README.md#instructions)) так и для PostgreSQL.
  
  Проблемы:
  - Если в данных есть запятые (адрес и т.д), то PostgreSQL прочитает их как новый столбец.
---

3. После преобразования **всех файлов** в один из описанных форматов, идёт создание таблиц куда будут импортироваться данные.
4. Определить названия таблиц, названия полей (столбцов) и их типов данных, а так же primary и foreign keys.
5. Создать необходимые таблицы с полями.
6. Импортировать данные в таблицы: **ПКМ** на таблицу, раздел **Import/Export**, далее указать путь к файлу, выбрать тип, указать разделитель, выполнить.
 - Если при выполнении возникла ошибка, то во вкладке **Processes** можно будет увидеть конкретную причину ошибки.

### Раздел 2 (создание диаграммы, сохранение скрипта БД)

**Диаграмма:**
1. Создаётся в pgAdmin 4 через нажатие ПКМ на созданную БД, далее **ERD For Database**, после находим на панели инструментов диаграммы **Download Image**, файл сохранится в формате **.png**.
 - В задании текущего года указано предоставить файл в формате **PDF**. Прямого экспорта **нет**, поэтому через **Adobe Acrobat** либо **Word**, преобразовываем **из .png в .pdf**.
---
**Скрипт БД:**
2. Создаётся через ПКМ по выбранной базе данных, раздел **Backup**, в открывшемся окне вводим название для файла, после нажимаем кнопку **Backup**
 - Если не выбирали место для сохранения, то файл БД сохранится в **Документы** по умолчанию.
