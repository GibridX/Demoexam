## Описание 

Данный проект представляет собой информационную систему для компании, занимающейся продажей обуви.  
Система разработана в рамках демонстрационного экзамена (ГИА ДЭ БУ) и реализует требования трёх модулей:

- Модуль 1 — разработка базы данных средствами СУБД  
- Модуль 2 — разработка алгоритма и создание приложения  
- Модуль 3 — разработка пользовательского интерфейса и реализация CRUD-функциональности  

> [!NOTE]
> Правила репозитория:
> 1. Вопросы, проблемы и рекомендации, прошу создавать во вкладке `Issues`, либо в лс Telegram, буду просматривать их и отвечать по возможности.
> 2. `Pull requests`, в проекте возможно использоваться не будут, но они настроены на запрет прямого `push` в ветку без подтверждения.

## Роли пользователей

| Роль | Доступ |
|------|--------|
| **Гость** | Просмотр товаров без поиска, фильтрации и сортировки |
| **Авторизованный клиент** | Просмотр товаров |
| **Менеджер** | Просмотр товаров с поиском, фильтрацией и сортировкой + просмотр заказов |
| **Администратор** | Полный доступ: добавление, редактирование, удаление товаров и заказов |

# Техническая часть 

## Используемые технологии

- **Язык программирования:** C#  
- **Платформа:** .NET 8.0 (Windows Forms)  
- **Библиотеки:** Npgsql.EntityFrameworkCore.PostgreSQL 8.0.11
- **СУБД:** PostgreSQL  

## Подключение к базе данных

```csharp
optionsBuilder.UseNpgsql(
"Host=localhost;Port=5432;Database=demoexam;Username=postgres;Password=pos234");
```

## Структура базы данных

### Таблицы:

- `users` — пользователи системы  
- `production` — товары  
- `orders` — заказы  
- `pickup_point` — пункты выдачи  

### Нормализация

База данных приведена к **третьей нормальной форме (3НФ)**:

- каждая таблица имеет первичный ключ;
- отсутствуют транзитивные зависимости;
- соблюдена ссылочная целостность;
- используется согласованная схема именования;
- реализованы первичные и внешние ключи.

## Особенности хранения данных

- В БД хранится **путь к изображению товара**
- Фотографии сохраняются в папке `Photos`
- Ограничение размера изображения — **300×200 px**
- При замене изображения старый файл удаляется
- Если изображение отсутствует — используется `picture.png`

## Модуль 2 — Разработка алгоритма и создание приложения

### Авторизация

- При запуске отображается окно входа
- Вход по логину и паролю (из БД)
- Возможность входа в роли гостя
- После авторизации отображается ФИО пользователя
- Реализован выход обратно на экран входа

### Отображение товаров

Каждая карточка товара содержит:

- Фото
- Наименование
- Категорию
- Описание
- Производителя
- Поставщика
- Цену
- Итоговую цену со скидкой
- Количество на складе

### Визуальные требования

- Скидка более 15% → фон `#2E8B57`
- Товар отсутствует на складе → голубой фон
- Старая цена:
  - перечеркнута
  - красного цвета
- Итоговая цена — черного цвета

## Запуск проекта

1. Установить PostgreSQL 16
2. Через pgAdmin4 или psql восстановить базу данных из бекап файла demoexam_backup.sql
4. Проверить строку подключения в `AppDbContext`
5. Запустить проект
>[!WARNING]
> Файл `demoexam_backup.sql` записан в кодировке UTF-8, которая по умолчанию не поддерживается Windows

Пошаговая инструкция:
1. Нажмите Win + R на клавиатуре, введите intl.cpl и нажмите Enter, чтобы открыть настройки региона.
2. Перейдите на вкладку «Администрирование».
3. Нажмите кнопку «Изменить язык системы...»
4. Установите галочку «Бета-версия: Использовать Unicode UTF-8 для поддержки языка во всем мире»
5. Нажмите «ОК» и перезагрузите компьютер.
